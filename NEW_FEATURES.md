# 自动化脚本生成平台 - 新功能说明

## 新增功能

### 1. 查看生成的脚本
- **位置**：流程编辑器顶部工具栏
- **按钮**：查看脚本
- **功能**：在对话框中显示生成的Python脚本代码
- **使用**：点击"生成脚本"后，点击"查看脚本"按钮查看代码

### 2. 下载脚本
- **位置**：流程编辑器顶部工具栏
- **按钮**：下载脚本
- **功能**：将生成的Python脚本下载到本地
- **文件名**：`automation_script_[时间戳].py`
- **使用**：点击"下载脚本"按钮，脚本会自动下载

### 3. 打包为可执行程序
- **位置**：流程编辑器顶部工具栏
- **按钮**：打包程序
- **功能**：将Python脚本打包成Windows或macOS可执行程序
- **平台选择**：
  - Windows (.exe) - 可在Windows上直接运行
  - macOS (.app) - 可在macOS上直接运行
- **使用流程**：
  1. 点击"打包程序"按钮
  2. 选择目标平台（Windows或macOS）
  3. 点击"打包"按钮
  4. 等待打包完成，自动下载可执行文件

## 工作流程

```
1. 创建项目
   ↓
2. 拖拽功能模块
   ↓
3. 编辑模块参数
   ↓
4. 点击"生成脚本"
   ↓
5. 查看/下载脚本（可选）
   ↓
6. 打包为可执行程序（可选）
   ↓
7. 在Windows/macOS上运行程序
```

## 技术实现

### 前端
- **查看脚本**：使用El-Dialog显示脚本内容
- **下载脚本**：使用Blob和URL.createObjectURL实现浏览器下载
- **打包对话框**：选择平台后调用后端API

### 后端
- **打包服务**：使用PyInstaller将Python脚本打包成可执行文件
- **API端点**：`POST /api/scripts/package`
- **请求参数**：
  ```json
  {
    "script": "Python脚本内容",
    "platform": "windows或macos",
    "name": "程序名称"
  }
  ```
- **响应**：二进制可执行文件

## 系统要求

### 打包功能
- 需要安装PyInstaller：`pip install pyinstaller`
- 需要安装脚本依赖：
  - selenium
  - playwright
  - apscheduler
  - requests

### 运行可执行程序
- **Windows**：Windows 7及以上
- **macOS**：macOS 10.13及以上

## 按钮说明

| 按钮 | 功能 | 前置条件 |
|------|------|--------|
| 生成脚本 | 根据流程配置生成Python脚本 | 至少添加一个模块 |
| 查看脚本 | 在对话框中查看生成的脚本 | 已生成脚本 |
| 下载脚本 | 下载Python脚本文件 | 已生成脚本 |
| 打包程序 | 打包为Windows或macOS可执行程序 | 已生成脚本 |
| 执行 | 在本地执行脚本 | 至少添加一个模块 |

## 注意事项

1. **打包时间**：首次打包可能需要较长时间（1-5分钟），请耐心等待
2. **依赖安装**：确保Python环境中已安装所有必要的依赖库
3. **文件大小**：打包后的可执行文件较大（50-200MB），包含Python运行时
4. **跨平台**：Windows上生成的exe只能在Windows上运行，macOS上生成的app只能在macOS上运行
5. **权限**：macOS上运行app可能需要授予执行权限

## 故障排除

### 打包失败
- 检查是否安装了PyInstaller：`pip install pyinstaller`
- 检查Python版本是否为3.8+
- 查看后端日志获取详细错误信息

### 脚本执行失败
- 检查浏览器驱动是否已安装（ChromeDriver、GeckoDriver等）
- 检查网址是否正确
- 查看执行日志获取详细错误信息

### 下载失败
- 检查浏览器是否允许下载
- 检查磁盘空间是否充足
